- `lock.fill` - Padlock (yellow)

### Dismiss Button
- `xmark` - X mark (white)

## Color Palette

### Error Severity Colors

**Critical (Red)**
- RGB: (255, 59, 48)
- Hex: #FF3B30
- Usage: Model, transcription, recording failures

**Warning (Orange)**
- RGB: (255, 149, 0)
- Hex: #FF9500
- Usage: Empty audio, quality issues

**Info (Yellow)**
- RGB: (255, 204, 0)
- Hex: #FFCC00
- Usage: Permission requests

**Accent (Blue)**
- RGB: (10, 132, 255)
- Hex: #0A84FF
- Usage: Action buttons

### Background Colors

**Banner Background**
- RGBA: (0, 0, 0, 0.9)
- Nearly opaque black

**Banner Border**
- RGBA: (error_color, 0.3)
- Semi-transparent error color

**Button Background**
- RGBA: (10, 132, 255, 0.8)
- Semi-transparent blue

**Text Colors**
- Title: White (255, 255, 255)
- Message: White 90% (255, 255, 255, 0.9)
- Caption: White 60% (255, 255, 255, 0.6)

## Accessibility

### Font Sizes
- Title: 13pt semibold
- Message: 11pt regular
- Button: 11pt medium

### Contrast Ratios
- White text on black background: 21:1 (WCAG AAA)
- Red/Orange/Yellow borders: Clearly visible
- Blue buttons: Sufficient contrast

### Keyboard Navigation
- Error banner is not keyboard-focusable (dismissible via new action)
- Buttons are keyboard accessible
- Escape key dismisses banner

## Summary

The error UI is designed to be:
1. **Non-intrusive**: Appears at top, doesn't block terminal
2. **Visually clear**: Color-coded by severity
3. **Actionable**: Clear buttons for next steps
4. **Informative**: Specific error messages with guidance
5. **Animated**: Smooth slide-down and fade-out
6. **Consistent**: Matches existing app design language
7. **Accessible**: High contrast, readable fonts
